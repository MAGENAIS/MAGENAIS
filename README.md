# Core Philosophy

Every architectural decision inside MAGENAIS follows these principles.

## 1. Browser First

The browser has become the world's most universal runtime.

MAGENAIS is designed to run directly inside modern browsers using open web technologies without requiring complex server infrastructure for the core platform.

---

## 2. Open Source

Knowledge grows through collaboration.

MAGENAIS is developed in the open and welcomes contributions from developers, researchers, educators, designers, and creators worldwide.

---

## 3. Provider Agnostic

No AI provider should dominate the platform.

Every provider is treated as a replaceable component connected through a unified abstraction layer.

Users choose providers.

Providers do not choose users.

---

## 4. Plugin Everything

Every major capability should be extensible.

If a feature can be implemented as a plugin...

it should be a plugin.

The core remains lightweight while the ecosystem grows indefinitely.

---

## 5. Workflow Driven

Artificial Intelligence is no longer about individual prompts.

Modern AI applications require reusable, visual, executable workflows capable of combining multiple providers, tools, and agents.

MAGENAIS is designed around workflows rather than isolated conversations.

---

## 6. Modular Architecture

Large software systems survive through separation of concerns.

Every subsystem inside MAGENAIS has a clearly defined responsibility, interface, and lifecycle.

Modules communicate through well-defined APIs rather than direct dependencies.

---

## 7. Privacy Respecting

User data belongs to the user.

MAGENAIS minimizes unnecessary data collection and enables local execution whenever possible.

Privacy is treated as a design requirement rather than an optional feature.

---

## 8. Local First

Cloud AI is powerful.

Local AI is empowering.

MAGENAIS embraces both.

The platform supports cloud providers, local models, self-hosted inference servers, and hybrid deployments through one unified architecture.

---

## 9. Community Powered

The long-term success of MAGENAIS depends on its ecosystem rather than its core.

Plugins.

Providers.

Themes.

Workflows.

Documentation.

Educational content.

Community innovation is considered the primary engine of growth.

---

## 10. AI Native

MAGENAIS is not traditional software with AI added later.

Artificial Intelligence is the foundation of every architectural decision.

Every subsystem is designed assuming AI will continue evolving rapidly over the coming decades.

# Browser-First Architecture

MAGENAIS is not a traditional AI web application.

It is an **AI Operating System** that runs entirely inside the browser.

Instead of relying on backend servers, MAGENAIS distributes intelligence across independent modules connected through an event-driven architecture.

The browser itself becomes the runtime.

Every subsystem communicates through standardized interfaces rather than direct dependencies.

This allows every component to evolve independently while maintaining system stability.

```
                   UI Layer
                       │
             Event Driven Messaging
                       │
                 Core Kernel
                       │
    ┌──────────┬────────────┬─────────────┐
    │          │            │             │
Providers   Plugins    Workflows     Projects
    │          │            │             │
Storage    Assets      Extensions     AI Models
```

The architecture emphasizes:

- Browser-first execution
- Modular design
- Zero vendor lock-in
- Progressive enhancement
- Offline capability
- GitHub Pages deployment
- Open standards
- Community extensibility

# Core Kernel

The Kernel is the heart of MAGENAIS.

Unlike traditional frameworks where components call one another directly, every subsystem communicates through the Kernel.

The Kernel is responsible for:

- boot sequence
- lifecycle management
- dependency initialization
- event propagation
- service registration
- plugin loading
- provider registration
- runtime orchestration

Nothing bypasses the Kernel.

Everything is registered.

Everything is discoverable.

Everything is replaceable.

```
Kernel

├── Boot Loader
├── Service Registry
├── Lifecycle Manager
├── Dependency Resolver
├── Event Dispatcher
├── Runtime Manager
├── Error Boundary
└── Extension Host
```

The Kernel makes MAGENAIS resilient to future growth.

# Event Bus

MAGENAIS uses an asynchronous publish-subscribe architecture.

Components never import one another directly.

Instead they publish events.

```
Provider Connected

↓

Event Bus

↓

UI Updates

↓

Workflow Engine

↓

Project Manager

↓

Plugins

↓

Logs

↓

Notifications
```

Benefits include:

- loose coupling
- scalability
- runtime extensibility
- easier debugging
- plugin compatibility
- future distributed execution

Typical events include:

```
provider.connected

provider.disconnected

provider.failed

workflow.started

workflow.finished

workflow.error

plugin.loaded

plugin.unloaded

asset.imported

asset.deleted

project.saved

project.loaded

chat.message

image.generated

audio.generated

video.generated
```

The Event Bus is one of the most important architectural foundations of MAGENAIS.

# Architecture at a Glance

MAGENAIS follows a layered, event-driven architecture designed for long-term scalability, maintainability, and extensibility.

```text
                           User Interface
                                  │
                    React + Vite + TypeScript
                                  │
                     ─────────────────────────
                           Extension API
                     ─────────────────────────
                                  │
                             Core Kernel
                                  │
     ┌──────────────┬──────────────┬──────────────┐
     │              │              │              │
 Event Bus    State Manager   Storage Manager   Router
     │              │              │              │
     └──────────────┴──────────────┴──────────────┘
                                  │
      ┌──────────────┬──────────────┬──────────────┐
      │              │              │              │
 Providers      Plugins      Workflows      Projects
      │              │              │              │
      └──────────────┴──────────────┴──────────────┘
                                  │
                       Browser Runtime APIs
```

Every layer communicates through stable interfaces rather than implementation details.
